# PRD：无障碍AI学习平台（暂定名）

---

## 1. 产品概述

- **产品名称**：无障碍AI学习平台（待正式命名，需品牌侧确认）  
- **一句话描述**：一款专为视障、听障人士设计的移动端优先、高可访问性（Accessibility-first）网页学习平台，支持语音+文字双模交互，帮助残障用户零门槛自学大模型基础操作与AI工具使用。  
- **核心价值**：  
  - ✅ **降低技术使用门槛**：将复杂AI操作转化为分步语音引导+高对比度可视化界面，适配低视力及全盲用户；  
  - ✅ **兼顾教学闭环**：内置教师管理后台（轻量级），支持教授远程发布课程、练习、进度追踪；  
  - ✅ **真·移动优先**：100%响应式网页应用（PWA），无需安装App，适配主流安卓/iOS手机（含读屏兼容），**不依赖电脑设备**。

---

## 2. 产品背景

### 痛点分析（基于访谈验证）
- **用户侧**：  
  - 视障/听障用户中 >85%仅持有智能手机，无电脑、无稳定宽带环境，现有AI教育平台（如Coursera、网易云课堂）严重缺乏WCAG 2.1 AA级可访问性支持（如屏幕阅读器兼容、键盘导航、高对比度模式）；  
  - 当前AI工具操作步骤冗长（如“打开浏览器→输入URL→登录→点击‘模型’→选择参数→粘贴提示词→提交”），对依赖语音/触控的用户极不友好；  
  - 市面无专为残障人群设计的AI实操训练产品，现有“无障碍”多停留在“能读出来”，未解决“如何一步步完成任务”的教学断点。  

- **教学侧**：  
  - 合作教授（核心内容提供方）需高效触达分散学员、同步推送练习、获取实操反馈，但当前依赖微信/电话，无法沉淀学习数据、无法验证操作正确性。

### 市场机会
- 政策驱动：《“十四五”残疾人保障和发展规划》明确要求“加快信息无障碍建设”，教育类无障碍数字产品属重点扶持方向；  
- 空白赛道：国内尚无聚焦“残障人群AI实操能力培养”的SaaS化学习平台；  
- 可扩展性：首期服务视障/听障群体，后续可延伸至认知障碍、肢体障碍等场景，模块化设计预留扩展接口。

## 5. 使用场景（典型）

### 场景1：低视力用户自学“用AI生成购物清单”
- 用户打开网页 → 开启高对比度模式（自动识别系统设置）→ 启动“语音引导” → 点击“开始任务” → 听到语音：“请对着手机说：‘帮我列一份周末超市购物清单，包含蔬菜、水果和日用品’” → 用户说完后，页面显示AI生成结果（大号字体+绿色高亮）→ 底部出现“复制”按钮（带振动反馈）→ 用户点击完成。

### 场景2：全盲用户通过TalkBack完成练习
- 用户用TalkBack滑动至“任务3：保存结果” → 听到语音：“双击此区域将结果保存到手机相册” → 用户双击 → 听到“已保存，共1张图片” → 页面顶部显示绿色对勾图标（同时触发短振动）。

### 场景3：教授远程教学管理
- 教授登录教师后台 → 上传新课程《用AI整理微信聊天记录》（含3段语音讲解+5个分步任务） → 设置发布时间 → 平台向所有学员静默推送（无通知） → 次日查看看板：发现23%学员在“选择聊天记录范围”步骤失败率高 → 调整该步骤增加1秒延迟提示。

## 6. 约束条件

| 类型         | 具体约束                                                                 |
|--------------|--------------------------------------------------------------------------|
| **技术限制**   | • 必须纯前端实现（无后端API依赖），所有AI调用走教授提供的第三方API密钥（平台不存储）<br>• 不支持WebRTC实时音视频（算力/带宽不足），语音讲解仅支持MP3上传播放<br>• 屏幕阅读器兼容需覆盖Android TalkBack（华为/小米/OPPO）及iOS VoiceOver（iPhone SE2及以上） |
| **资源限制**   | • 开发团队：2名前端（1主1备），0后端，0UI设计师（复用教授现有课件视觉规范）<br>• 测试资源：仅能邀请5名真实视障/听障用户参与UAT（教授协助招募）                             |
| **时间限制**   | • **MVP上线周期：72小时原型 + 7天开发 + 3天UAT → 总工期≤14天**<br>• 第3天交付可交互原型供教授试用并确认流程<br>• 第14天必须上线可公开试用版本（教授用于下周开班）                          |
